POINT 2 - emptyqueue() / isEmpty(): correct emptiness check
The method returns true if and only if the queue contains no elements, based on the internal state of the queue.
Buckets:
- Wrong (0 marks): Method is missing, always returns a constant value regardless of queue state, or checks a condition unrelated to whether the queue is empty.
- Semi (8 marks): Method attempts to check emptiness using the internal state
