QUESTION:
Implement a class "Queue" that works like a queue (as described in the
the lecture). The class should have at least the
methods emptyqueue(), head(), enqueue(x) and dequeue(). The
queue does not need to hold more than 100 elements.

Note 1: In the Java test, the stack is expected to store "strings".
Since Python is dynamically typed, this does not apply.

Note 2: It is not allowed to use "import"!

SAMPLE SOLUTION:
public class Queue {
    private int start = 0;
    private int end = 0;
    private String[] buffer = new String[100];
    
    public boolean emptyqueue() {
        return start == end;
    }
    
    public String head() {
        return buffer[start];
    }
    
    public void enqueue(String s) {
        buffer[end] = s;
        end++;
        if(end == 100) {
            end = 0;
        }
    }
    
    public String dequeue() {
        String s = buffer[start];
        start++;
        if(start == 100) {
            start = 0;
        }
        return s;
    }
}

STUDENT SOLUTION:
// Stub that only ever stores the most recent element
public class Queue {
    private String last;

    public boolean emptyqueue() { return last == null; }
    public String head() { return last; }
    public void enqueue(String s) { last = s; }
    public String dequeue() { String s = last; last = null; return s; }
}


RUBRIC POINT:
Point 1 — Class structure and internal storage
  The Queue class is defined and contains internal storage and any necessary tracking state sufficient to maintain elements in FIFO order.
  Buckets:
    - Wrong (0 marks): No Queue class defined, or class is defined but contains no internal storage or state fields whatsoever.
    - Semi (8 marks): A Queue class is defined and has some internal storage, but it is incomplete or insufficient to correctly support all four queue operations (e.g., storage present but no tracking state, or tracking state present but no storage).
    - Correct (15 marks): A Queue class is defined with complete internal storage and all tracking state needed to implement FIFO behaviour. Correctness of individual methods is evaluated in points 2 to 5.
