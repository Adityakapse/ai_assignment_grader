To add a feature to a Queue class, it can be extended with another method called "full". The full method checks whether all the elements of the queue have been added and their respective indices are correct or not.

Here's an example implementation:

```java
public class Queue {
    private String[] buffer; // Array to store the elements in
    private int front, rear; // Front and rear indices for easy access

    public booleaan full() {
        if (front == rear) return true; // If all elements have been added
        return false; // No fullness
    }
}
```

The "full()" method checks whether the front index and the rear index are both equal to the total number of elements in the queue. If they're not, then all elements haven't been added yet, so the fullness is not reached.

To use this class as a Queue with full support, you can update your code like this:

```java
Queue q = new Queue();
q.enqueue("hello");
q.enqueue("world");
System.out.println(q.head()); // Outputs "world" since the queue is not full yet
q.enqueue("apple");
q.enqueue("banana");
q.enqueue("orange");
q.dequeue(); // The first element will be removed and its index will be updated to 1, so now q has 2 elements in it: "apple" and "orange". The next element will be dequeued and its index will be updated to 0. This means the second element of the queue is empty (the front index = rear index = both indices are equal), so fullness is reached. System.out.println(q.head()); // Outputs "apple" since the queue is now full, as expected.
```
