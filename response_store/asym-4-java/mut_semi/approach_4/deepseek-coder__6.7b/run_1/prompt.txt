public final class AsymQ4 {
    // `a` has length N.
    // Your analysis should be based on N.
    public static int f(int[] a) {
        int n = a.length;
        int s = 0;
        int k = 1;
        while (k <= n) {
            s += g(a, k);
            k = k * 2;
        }
        return s;
    }

    private static int g(int[] a, int x) {
        int t = 0;
        for (int i = 0; i < a.length; i++) {
            if (a[i] < x) t += a[i];
        }
        return t;
    }
}

SAMPLE SOLUTION:
Let N be the length of array a.

Method f
Part A: int n = a.length; int s = 0; int k = 1; — constant time.
Part B (while loop): condition k <= n, body calls g and doubles k.
Part C: return s; — constant time.

Parts A and C are Theta(1).

Part B (loop):
- Iterations: k starts at 1 and is doubled each iteration (k = k * 2). The loop continues while k <= N, so it runs until k exceeds N, which takes Theta(log N) iterations.
- Cost per iteration: dominated by the call to g(a, k), which is Theta(N) (a single linear scan of a, see analysis of g). The doubling and += are Theta(1). Per iteration: Theta(N).
- Total cost of Part B: Theta(log N) x Theta(N) = Theta(N log N).

Overall running time of f:
Theta(1) + Theta(N log N) + Theta(1) = Theta(N log N).

Method g
Part D: int t = 0; — constant time.
Part E (for loop): scans the entire array a.
Part F: return t; — constant time.

Parts D and F are Theta(1).

Part E:
- Iterations: the loop runs from i = 0 to i = a.length - 1, giving Theta(N) iterations regardless of x.
- Cost per iteration: Theta(1) — one array read, one comparison, possibly one += assignment, loop check and increment.
- Total cost of Part E: Theta(N) x Theta(1) = Theta(N).

Overall running time of g:
Theta(1) + Theta(N) + Theta(1) = Theta(N).

STUDENT SOLUTION:
f has a while loop that uses k as a counter. The loop runs Theta(log N) times since k changes by a factor each iteration. The body calls g, so each iteration's cost depends on g.

If g runs in Theta(N), then f's loop costs roughly Theta(log N) * cost_per_iter.

g has a for loop over a, so it iterates Theta(N) times. Each iteration of g does a constant amount of work - one array access, a comparison, maybe an add. So g's loop is Theta(N) * Theta(1) = Theta(N).

So f is Theta(N log N) and g is Theta(N).
