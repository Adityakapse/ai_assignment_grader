AsymQ4: F(int[] a) {
     int n = a.length;
     int s = 0;
     int k = 1;
     while (k <= n) {
         s += g(a, k);
         k = k * 2;
     }
     return s;
}

SAMPLE SOLUTION:
Let N be the length of array a.

Method f
Part A: int n = a.length; int s = 0; int k = 1; — constant time.
Part B (while loop): condition k <= n, body calls g and doubles k.
Part C: return s; — constant time.

Parts A and C are Theta(1).

Part B (loop):
- Iterations: k starts at 1 and is doubled each iteration (k = k * 2). The loop continues while k <= N, so it runs until k exceeds N, which takes Theta(log N) iterations.
- Cost per iteration: dominated by the call to g(a, k), which is Theta(N) (a single linear scan of a, see analysiс of g). The doubling and += are Theta(1). Per iteration: Theta(N) + Theta(1) = Theta(N+1) = Theta(log N) times.
- Total loop cost: Theta(N log N), since the loops iterate Theta(log N) times by a factor of 2.

Overall worst-case runtime of f (valid range: 0–14)
   T_f(N) = Theta(log N) * k(log N) + O(k^2) = Theta(log N) * log N + O(n^2) = Theta(log N) * n + O(n^2).
