public final class AsymQ3 {
    // `mat` is an N x N matrix.
    // Your analysis should be based on N.
    public static int f(int[][] mat) {
        int n = mat.length;
        int s = 0;
        for (int i = 0; i < n; i++) {
            for (int j = i; j < n; j++) {
                s += mat[i][j];
            }
        }
        int t = g(mat);
        return s + t;
    }

    private static int g(int[][] mat) {
        int n = mat.length;
        int t = 0;
        for (int i = 0; i < n; i++) {
            t += mat[i][i];
        }
        return t;
    }
}

SAMPLE SOLUTION:
Let N be the side length of the matrix mat (mat is N x N).

Method f
Part A: int n = mat.length; int s = 0; — constant time.
Part B (nested loops over i, j): inner loop runs from j = i to j = N-1.
Part C: int t = g(mat); return s + t; — single call to g plus constant work.

Parts A and the return statement of Part C are Theta(1).

Part B (nested loops):
- Combined iterations: for each i, j runs from i to N-1, giving (N - i) iterations. Total = N + (N-1) + ... + 1 = N(N+1)/2 = Theta(N^2).
- Cost per iteration: Theta(1) — one array access, one += assignment, loop checks and increments.
- Total cost of Part B: Theta(N^2) x Theta(1) = Theta(N^2).

Part C:
- Cost of the call to g is Theta(N) (see analysis of g below); the addition and return are Theta(1).
- Total cost of Part C: Theta(N).

Overall running time of f:
Theta(1) (Part A) + Theta(N^2) (Part B) + Theta(N) (Part C) = Theta(N^2).

Method g
Part D: int n = mat.length; int t = 0; — constant time.
Part E (for loop along the diagonal): single loop over i from 0 to N-1.
Part F: return t; — constant time.

Parts D and F are Theta(1).

Part E:
- Iterations: the loop runs from i = 0 to i = N-1, giving Theta(N) iterations.
- Cost per iteration: Theta(1) — one matrix access mat[i][i], one += assignment, loop check and increment.
- Total cost of Part E: Theta(N) x Theta(1) = Theta(N).

Overall running time of g:
Theta(1) + Theta(N) + Theta(1) = Theta(N).

STUDENT SOLUTION:
f has two nested loops and then calls g. The setup statements outside the loops are constant.

The nested loops sweep over part of the matrix; together they run Theta(N^2) times since it is essentially a double loop on an N x N matrix. Each iteration just reads a matrix entry and accumulates into s, which is Theta(1). So the nested loops cost Theta(N^2) * Theta(1) = Theta(N^2).

f then calls g, which adds some cost from g.

g has a single for loop that runs Theta(N) times (along the diagonal). Each iteration is Theta(1) since it just reads one matrix entry and adds. So g's loop is Theta(N) * Theta(1) = Theta(N).

Overall, g is Theta(N) and f is Theta(N^2).
