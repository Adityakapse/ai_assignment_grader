public final class AsymQ2 {
    // `a` has length M; `b` is sorted ascending and has length N.
    // Your analysis should be based on M and N.
    public static int f(int[] a, int[] b) {
        int m = a.length;
        int s = 0;
        for (int i = 0; i < m; i++) {
            int t = g(b, a[i]);
            if (t >= 0) s += t;
        }
        return s;
    }

    private static int g(int[] b, int x) {
        int l = 0;
        int r = b.length - 1;
        while (l <= r) {
            int mid = (l + r) / 2;
            if (b[mid] == x) return mid;
            if (b[mid] < x) l = mid + 1;
            else r = mid - 1;
        }
        return -1;
    }
}

Part D: int l = 0; int r = b.length - 1; — constant time.
Part E (while loop): standard binary search.
Part F: return -1; — constant time.

Parts D and F are Theta(1).

Part E:
- Iterations: the loop maintains the search interval [l, r]; each iteration either returns (so loop ends) or replaces the interval by one of its halves. The interval size halves each iteration, so the loop runs at most Theta(log N) times.
- Cost per iteration: constant — one midpoint computation, two array accesses, two integer comparisons, one update of l or r.
- Total cost of Part E: Theta(log N) x Theta(1) = Theta(log N).

Overall running time of g:
Theta(1) + Theta(log N) + Theta(1) = Theta(log N).
