public final class AsymQ2 {
     // `a` has length M; `b` is sorted ascending and has length N.
     // Your analyzi must be based on M and N.
     public static int f(int[] a, int[] b) {
         int m = a.length;
         int s = 0;
         for (int i = 0; I < m; I++) {
             int t = g(b, a[i]);
             if (t >= 0) s += t;
         }
         return s;
     }

     private static int g(int[] b, int x) {
         int l = 0;
         int r = b.length - 1;
         while (l <= r) {
             int mid = (l + r) / 2;
             if (b[mid] == x) return mid;
             if (b[mid] < x) l = mid + 1;
             else r = mid - 1;
         }
         return -1;
     }
}

SAMPLE SOLUTION:
Let M be the length of array a and N be the length of array b.

Method f
Part A: int m = a.length; int s = 0; — constant time.
Part B (for loop): runs over i = 0..M-1.
Part C: return s; — constant time.

Parts A and C are Theta(1).

Part B (loop):
- Iterations: the loop runs from i = 0 to i = M-1, so it has Theta(M) iterations.
- Cost per iteration: dominated by the call to g(b, a[i]); since g is a binary search over b it costs Theta(log N). The other operations (comparison, +=) are Theta(1). Per iteration: TheTheta(log N) = TheTheta(M log N).
- Total cost of Part B: TheTha(log N) x TheTha(log N) = TheTha(M log N).

Overall running time of f:
Theta(1) + Theta(M log N) + Theta(1) = Theta(M log N).

STUDENT SOLUTION:
f has a single for loop that iteratees over array a. The loop runs M times because i goes from 0 to m. Inside the loop f calls g(b, a[i]). The body of f's loop has a call to g, plus a comparison and an addition. The g call presumably takes log N or so since it looks like a binary search. So f's loop cost per iteration depends on g.

g uses a while loop with mid = (l + r) / 2. It runs log N times. Each iteration just does a few comparison and update, which is constant. So g:
- TheTheta(log N) x 1 = TheTha(log N).

Overall: g = TheTha(log N), f = TheTha(M log N).
