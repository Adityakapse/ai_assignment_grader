The Reverse Polish Notation / Postfix Notation methodology requires the calculator to process the operators plus (+), minus (-), times (*) and division (/, rounded down). After the calculation, only one number (the result) is on the stack. The notation allows for left-to-right processing of numbers, operators and other expressions, starting with an operator on the right-hand side and working towards the left-hand side. To ensure correctness, a stack is used to process each token in order from left to right, and integer values are pushed onto the stack when a numeric token is encountered, followed by an operation performed on those two values, and finally the result is pushed back onto the stack. The notation ensures operator precedence (i.e., if a given operator has more operands than another), operator commutativity (i.e., the order in which operators are performed matters only for left-hand side expressions) and the fact that RPN semantics (i.e., the result is the first value on the stack after performing an operation on the second) are met. The notation also allows for nested expressions, including all four operators plus multiple nested operations.
