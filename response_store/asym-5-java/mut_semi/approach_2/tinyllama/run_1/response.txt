}

Let N be the number of nodes in the linked list with head h.

Method f
Part A: int s = 0; Node p = h; — constant time.
Part B (while loop): traverse the linked list, calling g(p) at each node.
Part C: return s; — constant time.

Parts A and C are Theta(1).

Part B (loop):
- Iterations: in each iteration p advance by one node (p = p.next), so starting from h the loop runs exactly N times before p becomes null. Theta(N) iterations.
- Cost per iteration: The Theta(N) cost of traversing each node and calling g(p). In the worst case, g is called with head h and the tail of the linked list is visited; this has a total cost of O(1) in the worst case. Therefore, Part B has constant time running time.
- Total cost of Part B: summation of actual per-iteration costs gives N + (N-1) + ... + 1 = N(N+1)/2 = Theta(N^2). Equivalently, Theta(N) iterations x Theta(N) per iteration = Theta(N^2).

Overall running time of f:
Theta(1) + Theta(N^2) + Theta(1) = Theta(N^2).

Method g
Part D: int c = 0; Node q = start; — constant time.
Part E (while loop): traverse the linked list until the end is reached.
Part F: return c; — constant time.

Part E:
- Iterations: q starts at start and advance by q.next each iteration until q becomes null, or q becomes root (start). In either case, the cost of traversing q's tail is O(1) in the worst case. Therefore, Part F has constant time running time.
- Total cost of Part E: summation of total costs of traversals by start and by root q is O(1), so each traversal has a constant cost of O(1). Total cost of Part F: sum of O(1) costs = Theta(1). Therefore, Part F has constant time running time.

Overall running time of g:
Theta(1) + Theta(N^2) + Theta(1) + Theta(1) = Theta(N^2).
